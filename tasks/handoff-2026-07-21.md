## How to run locally

```
npm install                      # repo root (workspaces)
cd apps/api
npx wrangler d1 migrations apply atlas-db --local
node seed/generate.mjs       && npx wrangler d1 execute atlas-db --local --file=seed/seed.sql
node seed/glove/generate.mjs && npx wrangler d1 execute atlas-db --local --file=seed/glove/glove-seed.sql
npx wrangler dev --port 8787     # API at http://127.0.0.1:8787
# smoke: curl http://127.0.0.1:8787/v1/companies/top-glove/results?period=quarter
cd ../web && NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8787 npm run dev
```

## Conventions

- Programs large enough for multiple PRs; every program has a doc in
  `management/programs/`. Plans in `management/plans/`. Reports in
  `management/reports/`.
- API responses mirror `apps/web/lib/types.ts` — change presenters and types
  together, never one side.
- Facts: positive magnitudes; sign is presentation. Missing data renders `—`,
  never fabricated. Every value traces to a `source` row.
